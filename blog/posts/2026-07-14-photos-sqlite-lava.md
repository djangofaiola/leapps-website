# Every Photos.sqlite Parser, Now in LAVA

If there is one database on an iOS device that rewards a careful examiner, it is Photos.sqlite. Locations, faces, favorites, what was hidden, what was trashed, what was shared, and now what was generated by AI. It is all in there.

The entire Photos.sqlite parser family in iLEAPP, 42 artifacts, is now converted to LAVA. Standardized UTC timestamps, guarded queries that survive schema changes between iOS versions, and full LAVA output for every one.

This is the work of Scott Koenig. If you do iOS photo forensics and you are not reading his research, fix that today. He documents the queries, the sources, and the reasoning behind them at [theforensicscooter.com](https://theforensicscooter.com). The iLEAPP parsers are the executable version of that research.

## What the family covers

The artifacts pull from three photo database sources: the main Photos.sqlite (PhDaPsql), the Syndication photo library (SyndPL), and the generative playground store (GenPlayPsql). Across those sources you get:

- Basic asset and album data
- Assets with valid locations
- Favorites, hidden, trashed, and burst assets
- GenAI-detected images
- People and detected faces
- Shared albums and iCloud Shared Photo Library assets and contributors
- iCloud shared links
- Asset internal resource data
- Camera, MediaAnalysis, SlideShow, and related plist configs
- Per-version reference tables from iOS 14 all the way through iOS 26

That iOS 26 reference table matters. Apple keeps changing the schema, and keeping these parsers current across every version is a real, ongoing effort.

## Why LAVA

Photos.sqlite generates a lot of rows. That is exactly where the old HTML report struggles. Load enough of them and the page can choke, hang, or crash outright. LAVA does not. It was built for data heavy output, and it hands you those rows in a fast, filterable, linkable interface instead of a static HTML page. When you are working faces against locations against timestamps, that difference is the difference between finding the story and missing it.
